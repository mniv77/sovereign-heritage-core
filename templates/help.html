Your Life, Sealed.

The Sovereign Heritage Core is a hardware-bound security protocol designed to keep your most sensitive data under your physical control.

1

The Hardware Handshake

Master Credentials

Enter your Master Email and Vault Password. These are the primary keys to your encrypted database.

Identity Challenge

If you use a new phone or browser, the system will detect "Unknown Hardware." You will be asked a security question to prove physical ownership.

Trusting Device

Once verified, the device is "Trusted" for one year, bypassing future challenges.

2

Navigation Protocol

1. Folders (Categories)

Select a category folder to unlock the record list inside.

2. Records (Registry)

Click a title to view details. A icon indicates a file is attached.

3. Details (Notes)

View secure text or download attached files (PDFs, images) directly.

3

Data Management

Adding New Records

Open a folder, click the (+) button, and enter your content. Use the "Attachment" field to seal files into the database.

Editor Hub / Information Root

The "Information Root" represents the Folder (Category) where your record lives. You can change this to move records between folders.

Deleting Records

Deletion is permanent. Because this is a sovereign system, there is no "Recycle Bin" on the server.

4

The Heritage Protocol (Backup & Restore)

Backup (Heritage Export)

Clicking **Backup** generates a JSON file containing your vault data.

Standard/Pro Tier: Portable JSON format. You are responsible for keeping this file in a safe, private location.
Elite Tier: Zero-Knowledge "Sealed" Blob. The file is itself encrypted and requires your master key to even read the structure.

Binary Archive (ZIP Protocol)

For users with large volumes of attached files, the **Binary Archive** protocol allows you to save the entire `note_attachments` table as a high-density ZIP file.

Technical Logic: This archive contains the raw binary content of every file in your vault, paired with a metadata manifest to ensure they re-attach to the correct records upon restoration.

Restore (Heritage Import)

If you get a new phone or need to move your data to a different device, use the **Restore** button. Select your previously saved JSON or ZIP archive to instantly re-inject your records and categories into the new node.

Security Note

For all users, "Restore" is only possible if you are logged in with the same master credentials. Standard tier users must be extra vigilant in protecting their backup files as they are not internally sealed. Binary ZIPs should be stored in a physical safe.
